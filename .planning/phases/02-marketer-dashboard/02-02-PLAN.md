### Part B — AlertList.tsx changes

The `AlertList` component at line 8 imports `computeMetrics` from `@/lib/marketing-mock`. This function expects campaign objects with `budgetSpent`, `clicks`, `impressions`, `conversions`, `revenue` fields. However, the API response now includes `kpiSnapshots` (the latest snapshot) which has these fields (`c.kpiSnapshots[0].clicks`, `c.kpiSnapshots[0].impressions`, etc.).

Modify the `computeMetrics` call on line 15 to use the kpiSnapshot data when available:

Change:
```typescript
const metrics = computeMetrics(c);
```

To:
```typescript
const snapshot = c.kpiSnapshots?.[0] || {};
const campaignWithMetrics = {
  ...c,
  impressions: snapshot.impressions ?? c.impressions ?? 0,
  clicks: snapshot.clicks ?? c.clicks ?? 0,
  conversions: snapshot.conversions ?? c.conversions ?? 0,
  revenue: snapshot.revenue ?? c.revenue ?? 0,
  budgetSpent: c.budgetSpent ?? 0,
};
const metrics = computeMetrics(campaignWithMetrics);
```

Also update the alert condition checks that reference campaign fields directly. On lines 23–31, the conditions use `c.budgetSpent`, `c.clicks`, `c.impressions`, `c.conversions`. Update these to use the snapshot values:

- Line 23: `c.budgetSpent > 500` → `campaignWithMetrics.budgetSpent > 500`
- Line 26: `c.clicks > 100 && c.conversions === 0` → `campaignWithMetrics.clicks > 100 && campaignWithMetrics.conversions === 0`
- Line 29: `c.ctr < 0.5 && c.impressions > 1000` — there is no `c.ctr` on the API Campaign model, nor `c.impressions`. Replace with: `Number(metrics.ctr) < 0.5 && campaignWithMetrics.impressions > 1000`

Important: The `campaignWithMetrics` variable should be computed ONCE per campaign (inside the `.map` callback) and then reused by both `computeMetrics()` and the alert condition checks. Ensure the variable is in scope for both uses.
