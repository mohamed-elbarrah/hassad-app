<objective>
Create the backend API surface that the marketer dashboard frontend needs: an aggregated campaign stats endpoint for KPI cards, and an enriched task list response that includes campaign data with the latest KPI snapshot for the AlertList.

Purpose: The dashboard currently shows hardcoded zeros for KPIs and the AlertList references `task.campaigns` which is not included in the API response. These two API changes provide the real data the frontend needs to wire up.

Output: Two new API capabilities — `GET /campaigns/my-stats` returning `{ activeCampaigns, totalBudgetUsed, avgRoas }`, and `GET /tasks/my?include=campaigns&deptName=MARKETING` returning tasks with embedded campaign data including the latest KPI snapshot.
</objective>

Create a new public method `async myStats(userId: string)` in `CampaignsService` (add it after the existing `findOne` method or before the private helper methods). The method must:

1. Query all campaigns where the campaign's task is assigned to this user. Use Prisma's nested filtering:
```typescript
const campaigns = await this.prisma.campaign.findMany({
  where: {
    task: {
      assignedTo: userId,
    },
  },
  include: {
    kpiSnapshots: {
      orderBy: { recordedAt: 'desc' },
      take: 1,
    },
  },
});
```

2. Compute from the result set:
   - `activeCampaigns`: `campaigns.filter(c => c.status === CampaignStatus.ACTIVE).length` — note: import CampaignStatus from `@hassad/shared` if not already imported
   - `totalBudgetUsed`: sum of `c.budgetSpent` across ALL campaigns (not just active ones — this represents total expenditure across all the marketer's campaigns)
   - `avgRoas`: average of the latest kpiSnapshot `roas` value across all campaigns that have at least one kpiSnapshot. If no campaigns have kpiSnapshots, return `0`. Compute as: `const campaignsWithRoas = campaigns.filter(c => c.kpiSnapshots.length > 0); const avgRoas = campaignsWithRoas.length > 0 ? campaignsWithRoas.reduce((sum, c) => sum + c.kpiSnapshots[0].roas, 0) / campaignsWithRoas.length : 0;`

3. Return `{ activeCampaigns, totalBudgetUsed, avgRoas }` as a plain JS object. NestJS serialization will wrap it in the envelope automatically.

Add the controller endpoint in `CampaignsController`:

```typescript
@Get('my-stats')
@RequirePermissions('marketing.campaigns.read')
async getMyStats(@CurrentUser() user: any) {
  return this.campaignsService.myStats(user.id);
}
```

Place this route **before** the `@Get(':id')` route in the controller file to avoid route collision (Express matches `my-stats` as an `:id` param if placed after). The exact insertion point is after the `findAll` route handler (around line 40) and before `findOne`.

The `@RequirePermissions` decorator value `'marketing.campaigns.read'` matches the existing permission key used on other campaign read endpoints in this controller.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | `findMine` deptName filter | mitigate | deptName is string-matched against `department.name` — Prisma parameterizes it, preventing SQL injection. No write operations use this filter. |
| T-02-02 | Information Disclosure | `findMine` campaigns include | mitigate | Campaigns are only returned for tasks assigned to the authenticated user (`assignedTo: userId`). No cross-user campaign leakage possible. |
| T-02-03 | Information Disclosure | `myStats` endpoint | mitigate | Stats are aggregated only from campaigns whose task is assigned to the authenticated user (`task.assignedTo` filter). No cross-user stats leakage. |
| T-02-04 | Denial of Service | `findMine` includeCampaigns | accept | `includeCampaigns=true` could return large payloads if a marketer has many campaigns. Accept for now — typical marketer workload is small (< 50 campaigns). |
</threat_model>
